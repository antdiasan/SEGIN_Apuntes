# Fortalecimiento del Sistema (Hardening Básico)

El **fortalecimiento del sistema**, también conocido como *hardening*, consiste en **aplicar medidas de seguridad activa** para reducir vulnerabilidades y reforzar la protección del equipo frente a ataques.

Su objetivo es mantener la **integridad, disponibilidad y confidencialidad** de los sistemas, siguiendo las recomendaciones de los fabricantes y buenas prácticas de seguridad.

Es una buena práctica buscar en Internet las medidas de *hardening* que pueden implantarse en el sistema informático e implementarlas.  
Existen organizaciones cuya misión es **clasificar posibles ataques y recomendar soluciones** a los administradores de sistemas y a los desarrolladores de software, facilitando así su labor.  
De este modo, no necesitan ser expertos en seguridad, sino simplemente seguir unas pautas de diseño o implementar cortafuegos con las reglas recomendadas por dichas entidades.

Los tres casos más significativos son:

- **Fundación OWASP (Open Web Application Security Project, [owasp.org](https://owasp.org))**: enfocada al desarrollo de aplicaciones, especialmente web.  
  Su trabajo se divide en proyectos, entre los que destaca el **Top Ten Project**, que incluye los diez ataques a aplicaciones web más comunes y cómo protegerse de ellos desde la fase de diseño del software.

- **Consorcio WASC (Web Application Security Consortium, [webappsec.org](https://webappsec.org))**: similar al anterior, pero dirigido en general a la **seguridad en la web**, abarcando tanto los sitios web como las organizaciones que los gestionan.

- **Organización sin ánimo de lucro CIS (Center for Internet Security, [cisecurity.org](https://cisecurity.org))**: dispone en su web de una sección denominada **Benchmarks**, desde la cual se pueden descargar (previo registro gratuito) **extensas guías para securizar** distintos sistemas operativos y aplicaciones de uso extendido (incluyendo Android, MySQL, SQL Server, Apache o Docker, entre muchas otras).  
  Cada recomendación de auditoría implementada en el sistema incrementará su puntuación final en el citado *benchmark*.

---

## Medidas básicas de fortalecimiento

### Actualización del sistema

Mantener actualizado el sistema operativo y las aplicaciones es una de las medidas más efectivas.  

Las **actualizaciones de seguridad** corrigen errores, mejoran el rendimiento y **cierran vulnerabilidades** que podrían ser aprovechadas por atacantes.

> Ejemplo:  
>   Instalar las actualizaciones automáticas de Windows o ejecutar en Linux:  
>   ```bash
>   sudo apt update && sudo apt upgrade
>   ```

No obstante, una actualización también tiene sus riesgos: puede dejar el sistema inestable o inservible a causa de alguna incompatibilidad del nuevo software con otro existente en el mismo equipo o con el hardware subyacente.  
Muchos administradores prefieren seguir el criterio de *“lo que funciona, mejor no tocarlo”*, pero esta filosofía, en seguridad informática, es inherentemente peligrosa.

!!! warning "Recuerda"
    Siempre deben instalarse las actualizaciones de seguridad.  
    Sin embargo, en **servidores sensibles**, antes de hacerlo se deben tomar medidas de recuperación ante un posible problema de incompatibilidad: copias de seguridad o instantáneas del sistema.

---

### Eliminación de servicios innecesarios

Cada servicio en ejecución representa un posible punto de ataque.  

Desactiva aquellos que no sean esenciales (por ejemplo, Bluetooth, servicios de escritorio remoto o servidores locales no utilizados).

---

### Gestión segura de usuarios y contraseñas

- Crea cuentas con **privilegios mínimos**.  
- Cambia las contraseñas por defecto.  
- Utiliza **contraseñas seguras** (combinando letras, números y símbolos).  
- Evita usar la cuenta de administrador para tareas cotidianas.

---

### Configuración del firewall

El **firewall** actúa como una barrera de protección frente a conexiones no autorizadas.  

Debe estar siempre activado y configurado para permitir solo el tráfico necesario.

!!! tip "Ejemplo práctico"
    En Windows, accede a  
    *Panel de control → Firewall de Windows Defender → Configuración avanzada*  
    y revisa las reglas de entrada y salida activas.

## Herramientas útiles

| **Herramienta** | **Sistema** | **Función principal** |
|------------------|-------------|------------------------|
| **Windows Defender / Firewall** | Windows | Protección en tiempo real y filtrado de red |
| **UFW (Uncomplicated Firewall)** | Linux | Configuración sencilla del cortafuegos |
| **chkrootkit / rkhunter** | Linux | Detección de rootkits en el sistema |

---

## Tabla de buenas prácticas

| **Acción** | **Objetivo** | **Beneficio** |
|-------------|---------------|---------------|
| Actualizar el sistema y aplicaciones | Cerrar vulnerabilidades conocidas | Mayor seguridad y estabilidad |
| Desactivar servicios innecesarios | Reducir puntos de ataque | Menor riesgo de intrusión |
| Usar cuentas con mínimos privilegios | Limitar daños en caso de infección | Protección frente a errores o malware |
| Activar y configurar el firewall | Controlar conexiones entrantes/salientes | Bloqueo de accesos no autorizados |
| Usar contraseñas seguras | Aumentar la confidencialidad | Previene accesos indebidos |

---

    Realiza una **lista de comprobación (*checklist*)** con 10 medidas básicas de seguridad aplicadas en tu propio equipo.  
    Comprueba qué configuraciones ya estaban activas y cuáles has mejorado.
