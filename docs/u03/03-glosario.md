# Glosario

### Analizador de paquetes (*sniffer*)
Aplicación que captura todo el tráfico que pasa por la interfaz de red del equipo informático y permite analizar el contenido de todos los paquetes que entran y salen de la misma.

---

### Descarga no consentida (*drive-by download*)
Descarga realizada sin el consentimiento del usuario.

---

### Dirección IP (*Internet Protocol*)
Valor de 32 bits que identifica a una interfaz de red dentro de una red que utiliza el protocolo IP.  
El agotamiento de las direcciones IPv4 ha provocado la creación de las **direcciones IPv6**, de 128 bits.  
Las anteriores han pasado a denominarse **IPv4**.

---

### Dirección MAC (*Media Access Control*)
Identificador de 48 bits que se asigna a todos los dispositivos de red, garantizando que no existan dos dispositivos con la misma MAC.  
Esto se logra porque la **IEEE** asigna los primeros 24 bits a cada fabricante del mundo.

---

### Dominio
Conjunto de ordenadores conectados en red y con gestión centralizada.

---

### Hardening
Proceso mediante el cual se aumenta la seguridad de un sistema informático —como un servidor, equipo o red— reduciendo su superficie de vulnerabilidad.

---

### Macro
Término que se emplea para hacer referencia a pequeñas cantidades de código ejecutable incrustadas en documentos no ejecutables, como por ejemplo los de Word o Excel.  
Las macros se ejecutan en respuesta a acciones del usuario sobre dichos documentos (por ejemplo, antes de guardar).

---

### Puerta trasera (*backdoor*)
Método para iniciar sesión en un sistema eludiendo el proceso de autenticación.

---

### Puerto (*port*)
Conector virtual utilizado para establecer conexiones de red mediante los protocolos **TCP** o **UDP**.  
Gracias a los puertos, un mismo servidor puede ofrecer varios servicios simultáneamente.  
La organización **IANA** (*Internet Assigned Numbers Authority*) se encarga de establecer los puertos utilizados por servicios estándar, como por ejemplo el **22** para **SSH** o el **80** para **HTTP**.

---

### Servidor DHCP (*Dynamic Host Configuration Protocol*)
Servidor encargado, dentro de una red IP, de **asignar direcciones IP** a los equipos y dispositivos que las solicitan al iniciarse, evitando así la configuración manual (y propensa a errores) de cada equipo por separado.

---

### TCP (*Transmission Control Protocol*)
Protocolo de transporte que se sustenta sobre el protocolo IP.  
Se caracteriza por garantizar conexiones fiables mediante la repetición de paquetes mal recibidos (o no recibidos), su ordenación y el control de velocidad.

---

### UDP (*User Datagram Protocol*)
Protocolo de transporte que se sustenta sobre el protocolo IP.  
A diferencia de **TCP**, **UDP** no garantiza una conexión fiable; simplemente permite el envío de paquetes de datos (llamados *datagramas*) hasta el destino, sin establecer una conexión previa ni controlar su correcta recepción.
