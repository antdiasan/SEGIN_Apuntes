# Qué es un proxy

Un **proxy** es un sistema que actúa como intermediario entre un usuario y los servicios de Internet.  
Cuando se utiliza un proxy, las comunicaciones no se realizan directamente entre el equipo del usuario y el destino final, sino que pasan previamente por este intermediario.

El uso de proxys permite controlar, gestionar y, en algunos casos, mejorar la seguridad y la privacidad de las comunicaciones.
Pero, también sus contrapartidas:

- Puede representar un cuello de botella en la red local,
- La función caché y la de análisis del tráfico pueden entrar en conflicto con la protección de datos personales,
- La función cache supone el riesgo de entregar a los usuarios versiones anticuadas de las webs que visitan,
- Es necesario configurar los equipos clientes para que utilicen el servidor proxy, habitualmente se configura en el puerto 8080, aunque puede utilizarse otros.

Un **proxy transparente** es aquel que funciona sin la necesidad de configurar los equipos cliente. Para ello intercepta el tráfico de los puertos 80/443 y lo redirige al software proxy. Dicha redirección sólo puede producirse en el *gateway* de la red. 
Suele implementarse mediante reglas en el cortafuegos del gateway.

Existe un protocolo llamada **SOCKS** que puede ser utilizado para realizar conexiones TCP de cualquier protocolo a través de un proxy (también UDP en SOCKS5). Sin embargo, solo los cliente preparados para conectar a través de un proxy (navegadores, clientes email, FTP, etc) están preparados para hacerlo mediante un proxy SOCKS. 
Existen aplicaciones llamadas **clientes SOCKS** que, una vez instaladas en un equipo, permiten que toda la comunicación de equipo se realice a través del servidor (proxy).

---

!!! info "Configuración técnica"
    Aunque un servidor proxy puede configurarse en cualquier puerto, el estándar industrial más utilizado es el **8080**. Como técnicos, cuando configuréis manualmente un navegador o una aplicación para salir a través de un proxy, este será el puerto que deberéis indicar junto a la dirección IP del servidor.


!!! abstract "Idea clave"
    Un proxy actúa como intermediario entre los usuarios e Internet, permitiendo controlar, filtrar y registrar el tráfico.


---

## Funcionamiento básico de un proxy

En una comunicación normal, el equipo del usuario se conecta directamente a un servidor en Internet.

Cuando se utiliza un proxy, la petición se envía primero a este sistema intermedio, que se encarga de reenviarla al destino y devolver la respuesta al usuario.

De este modo, el servidor externo no se comunica directamente con el equipo del usuario.

---

## El proxy como intermediario

Al actuar como intermediario, el proxy puede observar y gestionar el tráfico que pasa a través de él.

Esto permite aplicar políticas de control, como limitar accesos, registrar conexiones o filtrar determinados contenidos.

El proxy se convierte así en un punto central de control dentro de la red.


!!! note "Nota aclaratoria"
    El proxy no es un cortafuegos, aunque ambos pueden complementarse dentro de una estrategia de seguridad.

---

## Tipos generales de proxys

Existen distintos tipos de proxys según su función y su forma de uso.

Algunos se utilizan para controlar el acceso a Internet en redes internas, mientras que otros se emplean para mejorar la privacidad o el rendimiento de las conexiones.

La elección del tipo de proxy depende de las necesidades de la red y del objetivo que se persiga.

---

## Proxys y privacidad

El uso de un proxy puede contribuir a mejorar la privacidad al ocultar la dirección IP real del usuario frente a los servicios externos.

Sin embargo, el propio proxy tiene acceso al tráfico que gestiona, por lo que debe ser un sistema confiable y correctamente administrado.

La privacidad depende tanto de la tecnología utilizada como de quién la gestiona.

---

!!! warning "Riesgo de seguridad"
    Utilizar proxys de origen desconocido puede comprometer la **seguridad y la privacidad** de las comunicaciones.
